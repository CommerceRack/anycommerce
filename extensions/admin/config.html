Optional Configuration

Paypal offers two different methods of authentication for api requests.
1st party: where you create your own unique credentials
3rd party: where you give paypal_api1.zoovy.com access to your account (recommended, easier)

If you have already granted API access to paypal_api1.zoovy.com on PayPal.com, then leave all the fields below empty of the configuration will NOT work.

Zoovy must provide a shipping "ESTIMATE" to Google prior to initiating a transaction, and before the destination zip code is known. This Zip code will be used for providing an estimate to Google.

We recommend you put in a far away zip code, because the estimate will be used in the event that Google Checkout is not able to obtain shipping rates from Zoovy once the customer has provided their shipping address.

You may safely set this to 00000 if you use fixed price shipping and have no suppliers who use zone-based shipping.
In cases were there is a communication error the "Do Not Audit" setting will use whatever rate Google checkout quoted the customer. This is the most common setting since it creates the lowest amount of work for the seller, although it could also cause orders to occasionally have more or less than the correct shipping rates due to timeout issues with merchant calculations.

Google requires that Zoovy pass shipping rate estimates for all international countries before the actual shipping address is known. This can unfortunately lead to very "bad" rate quotes for merchants who wish to use zone based shipping. Only enable this if you use fixed price international shipping.

Use the format:
http://www.domain.com/pixel.gif?x=1&y=2&orderID=[order-id]&totalCost=[order-total]

The values in [brackets] are the parameterized url keys/values. A complete list can be found here:
http://code.google.com/apis/checkout/developer/checkout_pixel_tracking.html
